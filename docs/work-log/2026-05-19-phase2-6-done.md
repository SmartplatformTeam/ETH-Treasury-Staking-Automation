## 검증된 사실

1. **per-node 격리**: 한 Node 의 fetch 실패가 다른 Node 처리에 영향 없음 — try/catch 가 per-node 단위.
2. **transactional 갱신**: Node update + NodeHealthSnapshot insert 가 `prisma.$transaction` 으로 atomic. 한쪽 실패 시 둘 다 rollback.
3. **worst-of host status**: 같은 호스트의 여러 Node 중 가장 심각한 status 로 OperatorHost 갱신. STATUS_SEVERITY 매핑.
4. **timeout 안전**: AbortSignal.timeout(5_000) 으로 Beacon 무응답이 worker 를 block 하지 않음.
5. **격리 원칙 유지**: lighthouse 컨테이너에 write 없음 — 순수 read API. control-plane DB 에만 기록.
6. **alerts auto-wired**: insights.service 가 이미 unhealthy Node/OperatorHost 합성. 데이터만 들어가면 자동 표시.
