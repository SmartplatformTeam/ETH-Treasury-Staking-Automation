## 검증된 사실

1. **API 컨테이너 내부에서 ansible 실행이 정상 동작**. child_process.spawn 경로가 stdout/stderr를 `AutomationRunEvent`로 stream하고 종료 시 status/exitCode를 갱신. fire-and-forget 비동기.
2. **AUTOMATION_REPO_ROOT 자동 resolve가 컨테이너에서 정상**. `process.cwd() = /app` → `/app/infra/ansible/playbooks/full-operator-mvp.yml` 존재 확인 → repoRoot resolve.
3. **`team-server-mvp.yml` idempotent 재배포가 동작**. 기존 떠 있던 control-plane 위에 코드 변경(verify-baseline + Dockerfile ansible)을 그대로 반영. ok=29 changed=7 failed=0.
4. **격리 원칙 유지**. 팀 서버의 기존 obol 운영(node0~3-charon, vc-*, web3signer-*, mev-boost, grafana, prometheus)은 일체 무관. control-plane compose project만 affect.

## Phase 1로 잠긴 인터페이스 (Phase 2+가 그대로 재사용)

- API 컨테이너 안에서 `ansible-playbook -i <inventory> --limit <host> <playbook> --extra-vars <json>` 형식으로 호출
- safe 작업은 approvalId 없이, execute 작업은 approvalId + cluster/host/policy 매칭 검증
- 상태 흐름: QUEUED → RUNNING → SUCCEEDED/FAILED/CANCELLED
- 로그는 sequence + stream(STDOUT|STDERR|SYSTEM) + redacted flag로 DB 저장
- inventoryRef 컬럼: 실 inventory path 또는 `inline:localhost` (host-agnostic 작업용)

## 보안 정리

- 평문으로 한 번 들어온 SSH password는 어떤 .md/git/inventory 파일에도 기록되지 않음.
- SSH key 인증으로 전환 완료 → 이후 password 입력 불필요.
- 권장: 공유된 그 평문 password는 회의 후 rotation.

## 남은 follow-up

- ANSIBLE_BECOME_PASS env var는 사용하지 않았음 (interactive `--ask-become-pass`로 통과).
- 팀 서버에 sudoers NOPASSWD 또는 ansible-vault 도입은 Phase 2 자동화 빈도 늘어날 때 결정.
- redactor 정책 검증: 현재 verify-baseline은 민감 출력이 없지만 다른 op에서 secret이 stdout으로 새지 않는지 (`log-redactor.ts`) Phase 2에서 별도 확인.
