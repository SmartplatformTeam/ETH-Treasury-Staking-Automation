## 1. apps/api automation/runs

이미 구현된 부분:

- `apps/api/src/modules/automation/automation.controller.ts:26-59`
  - `POST /automation/runs` (createRun), `GET /automation/runs` (list), `GET /automation/runs/:id`, `POST /automation/runs/:id/cancel`
  - `@RequirePermissions("inventory:read")` guard + role-based cancellation (ADMIN/INFRA_OPERATOR)
- `apps/api/src/modules/automation/automation.service.ts:137-211`
  - createRun이 cluster/host 존재 검증 → `assertCreateAllowed()`로 safe vs execute tier 분기 → execute는 `approvalId` 필수
  - `AutomationCommandBuilder`로 ansible 명령 빌드 → DB QUEUED 레코드 생성 → `this.runner.start(run.id, command)` fire-and-forget
- `apps/api/src/modules/automation/automation-runner.service.ts:13-110`
  - child_process spawn으로 ansible-playbook 실행 → stdout/stderr를 `AutomationRunEvent`로 stream → 종료 시 status/exitCode 갱신
- DTO: `CreateAutomationRunDto`에 operation enum, clusterId, hostId, dryRun, approvalId, extraVars whitelist (cluster_name, host_name, deployment_path 등)

빠진 부분: 거의 없음. 단 **`AutomationOperation` enum에 `verify-baseline`이 들어 있는지** S2에서 확인 필요.

## 5. 컨테이너 & mount

- `infra/docker/control-plane.Dockerfile`: base `node:20-bookworm-slim`, ca-certificates + openssl만 설치. **ansible 바이너리 없음.**
- 전체 레포를 COPY해서 빌드 → `infra/ansible/`, `infra/obol-cdvn/scripts/`가 컨테이너 안에 존재
- `AUTOMATION_REPO_ROOT` env로 repo 루트 resolve. inventory도 컨테이너 안 path 사용 가능.

## 결론: 빠진 것 2개

1. **Dockerfile에 ansible-core + 의존성 설치 필요** (`ansible-core`, `python3`, optional `python3-pip`).
2. **`verify-baseline.yml` playbook 신규 생성 필요** — host 무관/inventory 무관으로 `infra/obol-cdvn/scripts/verify-baseline.sh`를 control plane host(=API 컨테이너 localhost) 안에서 직접 실행하는 1-task playbook.
   - 부수적으로 `AutomationOperation` enum과 `AutomationCommandBuilder`의 operation→playbook 매핑에 `verify-baseline` 항목을 추가해야 할 수 있음 (S2에서 확인).

이게 끝나면 S5 (Web 버튼)는 거의 그대로 동작 가능 — controller/service/model이 다 갖춰져 있기 때문.

## S2 어댑터 설계 (재정의: 빈틈만 메움)

- 입력 enum: `verify-baseline` operation 추가, playbook 경로 `infra/ansible/playbooks/verify-baseline.yml`, inventory는 localhost 단일 호스트 사용 또는 inventory 불필요한 `--connection=local`
- safe tier로 분류 → approvalId 불필요
- extraVars 불필요 (verify-baseline은 인자 없음)
- 단위 verify: `docker compose run api ansible-playbook --syntax-check infra/ansible/playbooks/verify-baseline.yml`

## 다음 단계 (Phase 1 step 분해 재조정)
