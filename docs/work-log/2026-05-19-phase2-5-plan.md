## Phase 2-5 범위 (한 줄)

`/automation` 페이지에서 execute tier 작업 선택 시 일치하는 APPROVED approval 을 자동으로 드롭다운에 채워서 operator 가 두 페이지를 왔다 갔다 하지 않게.

## 잠긴 결정 (사용자 confirm)

1. **approval 입력 UX**: execute op 일 때만 dropdown (cluster+host+operation 일치하는 APPROVED 만 자동 채움). safe op 일 때는 approval 섹션 자체를 숨김.
2. **매칭 approval 없음 처리**: submit 버튼 비활성화 + 인라인 안내 + `/approvals?prefill=<base64 json>` 링크. operator 가 그 링크 따라가면 `/approvals` 의 생성 form 이 자동으로 채워진 상태로 펼침.
3. **Cluster → Host cascade**: 추가. /approvals 의 create form 과 동일 패턴 (cluster 선택 → 해당 cluster 소속 host 만 노출).
4. **GET /v1/approvals API filter**: `clusterId`, `hostId`, `automationOperation` 세 가지 query parameter 추가.

## 잠긴 결정 (조사 기반 추가)

5. **client component 전환**: 현재 `/automation` page 는 server component + server action FormData. cascade + 동적 dropdown 을 위해 form 부분을 `'use client'` component (`automation-form.tsx`) 로 분리. server action 은 그대로 호출.
6. **prefill 인코딩**: `?prefill=<base64-encoded JSON>` 형식. JSON 은 `{ policyType: "ROLLOUT", clusterId, hostId, automationOperation }`. base64 로 URL-safe 하게 인코딩. /approvals 페이지가 prefill 받으면 form 자동 expand + 값 prefill.
7. **API filter 검증**: `clusterId`/`hostId`/`automationOperation` 모두 optional. 빈 문자열은 무시. 검증 실패(잘못된 enum 값) 는 400.
8. **safe op 일 때 approval 필드**: form 의 approval 섹션을 DOM 에서 완전 제거 (display:none 보다 unmount). FormData 에 approvalId 자동 미포함.
9. **execute op 권한 분기 (UI)**: INFRA_OPERATOR/ADMIN 만 자동화 호출. 현재 UI 분기 없음 — 이번에 같이 처리 (form 자체를 mount 안 함). server-side 가드 `assertCreateAllowed` 는 유지 (defense in depth).
10. **Recent Runs / Run Detail 패널**: 그대로 유지. 변경 없음.

## 위험과 대응

| 위험 | 대응 |
|---|---|
| Client component 가 매번 fetch 하면 race condition | useEffect + AbortController 또는 첫 로드 시 한꺼번에 가져오고 client filter |
| Prefill JSON 변조 시 의도하지 않은 필드 노출 | server-side parseCreateApprovalDto 가 다시 검증. UI 만 prefill 처리, 실제 제출은 동일 검증 |
| approval dropdown 에 너무 많은 옵션 | 첫 로드 APPROVED 만 가져옴. 추후 페이징 follow-up |
| Cluster→Host cascade 가 운영 cluster 와 sandbox cluster 모두 표시 | 의도된 동작. operator 가 sandbox 명시적 선택 |
| Role 없는 사용자가 직접 server action 호출 | server-side assertCreateAllowed 가 ForbiddenException. UI 분기는 UX 만 |
