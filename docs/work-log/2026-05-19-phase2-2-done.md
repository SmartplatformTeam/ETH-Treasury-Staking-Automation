## 잠긴 architecture/규칙 (Phase 2-3+ 재사용)

1. **Worker scheduler**: `setInterval` 기반 단일 process, BullMQ 미도입 (실패 격리는 try/catch per validator). 동시 실행 늘면 worker 별도 분리 검토.
2. **Beacon endpoint**: control-plane이 production beacon network에 read-only join (`hoodi-node-network` external). validator state만 호출, write 안 함. 베어메탈 이관 시 inventory의 `control_plane_beacon_*` 변수만 변경.
3. **BigInt 직렬화**: API 응답에서 BigInt → string으로 명시 변환 (`balanceGwei.toString()`). web layer에서 `gweiStringToEthDisplay` 로 ETH 표시.
4. **FAR_FUTURE_EPOCH 처리**: parser에서 `"18446744073709551615"` 감지 → null. schema의 epoch 컬럼은 모두 `BigInt?`.
5. **DataTable layout**: `table-fixed` + `<colgroup><col style={width}/></colgroup>` + cell inner div의 inline `text-align: center; white-space: nowrap` + `text-ellipsis` + `overflow-hidden`. inline style이 CSS cascade/browser cache 영향 안 받음.
6. **Panel layout**: dashboard/automation에서 `xl:grid-cols-*` 대신 `flex flex-col gap-6` (single col stack). panel이 main 영역 full width로 column이 충분히 분배됨.

## 검증된 사실

1. **5분 cycle로 balance 변화 관찰**: Parataxis active validator의 balance가 cycle마다 약 26000~50000 gwei 증가. 실제 staking reward 누적 그대로 표시.
2. **Beacon network 격리**: control-plane이 `hoodi-node-network` external join. lighthouse 컨테이너 안 건드림, 기존 obol 운영 4-5주 무영향.
3. **DataTable column 강제 분배**: column.width prop 명시한 column은 보장된 너비 유지 (chip이 panel 밖으로 빠지지 않음).
