### 로컬 환경 구성 산출

신규 파일: [infra/docker/compose.dev.yml](../../infra/docker/compose.dev.yml) — postgres 16 + api + web 3 service, dev 전용. `eth-staking-app:dev` 이미지 (2.01GB, ansible 2.14.18 포함).

띄우는 명령:

```bash
docker compose -f infra/docker/compose.dev.yml up -d postgres
docker compose -f infra/docker/compose.dev.yml run --rm api pnpm db:push
docker compose -f infra/docker/compose.dev.yml run --rm api pnpm db:seed
docker compose -f infra/docker/compose.dev.yml up -d api web
# API: http://localhost:4000  |  Web: http://localhost:3000
```

호출 예 (RBAC stub 헤더 사용):

```bash
curl -X POST -H "x-eth-user-role: ADMIN" -H "Content-Type: application/json" \
  -d '{"operation":"VERIFY_BASELINE","clusterId":"<id>","hostId":"<id>","dryRun":false,"extraVars":{}}' \
  http://localhost:4000/v1/automation/runs
```

## 팀 서버 재배포 메모

- `team-server-mvp.yml` 호출 시 기존 obol 운영 컨테이너(node0~3-charon, vc-*, web3signer-*, mev-boost, grafana, prometheus)는 무관 — control-plane compose project만 stop/build/up.
- DB persistent volume 유지로 seed upsert가 idempotent 통과.
- 첫 빌드라 ansible apt 설치 단계가 추가되어 빌드가 길어졌지만 동일 이미지가 재사용되므로 다음 호출은 빠름.
- inventory: `~/.config/eth-staking-ansible/team-server/hosts.yml` (사용자 Mac, repo 밖, commit 없음).

## 격리 원칙 준수

- verify-baseline은 baseline mirror 무결성만 검증 (host-agnostic).
- 팀 서버의 기존 obol 4 clone 디렉토리/컨테이너에 접근/수정 0.
- 새 EL/CL/charon 컨테이너 띄우지 않음.
- I/O 부하 0 (디스크 read만).
- 따라서 격리 cluster name `team-sandbox-obol-a`와 무관하게 안전.

## 보류 항목과 의존성

S5'/S6' verify는 다음 환경 의존:

1. **Docker Desktop 가동** — `docker compose build api`가 통과해야 ansible 설치 확인.
2. **PostgreSQL** — control_plane Ansible role이 compose mode로 띄우거나, 별도 dev DB 가동 (`pnpm db:push` + `pnpm db:seed`로 cluster/host fixture 필요).
3. **Fixture cluster/host 존재** — UI form이 cluster/host 강제 dropdown. seed에 sandbox cluster 또는 placeholder가 있어야 verify-baseline 호출 가능. (verify-baseline은 builder가 host 값을 ignore하지만, DTO/service가 존재 확인.)
4. **팀 서버 SSH 정보** (S6'만) — IP, user, deployment path.
